Fast Fashion Impact
Visualization Documentation

Project Documentation

Interactive visualization of the environmental, labor, and social costs of fast fashion, built with D3.js.

Dataset

The dataset used is The True Cost of Fast Fashion Impact by Khushi Yadav on Kaggle (MIT license). It contains 3,000 records across 25 attributes covering brand identity, production volume, environmental impact (carbon, water, landfill), labor conditions (wages, working hours, child-labor incidents), social-media attention, sentiment, and sustainability / ethics scores.

Design Process

Initial Exploration

The project began with examining the dataset columns to identify natural question clusters: environmental cost vs. production scale, labor conditions vs. brand, and sustainability scores vs. social-media sentiment. Early paper sketches explored a small-multiples layout (one chart per brand), a parallel-coordinates view, and a single scatterplot with dropdown-driven axes.

Iteration

The small-multiples approach was discarded because 6+ brands times 20+ numeric columns made the grid unwieldy. Parallel coordinates were explored but proved difficult to brush precisely with many overlapping polylines. The final design settled on a brushable scatterplot with user-selectable axes and color encoding — a flexible yet focused layout that lets the viewer pose and answer their own comparison questions.

Design Rationale

Chart Type — Brushable Scatterplot

A scatterplot was chosen because the core questions involve correlations between two continuous variables (e.g., production volume vs. carbon emissions). The brush interaction (D3 d3.brush()) lets users select a rectangular region and instantly see summary statistics for that subset, supporting rapid hypothesis testing without leaving the page.

Axis Dropdowns

With 21 numeric attributes, hard-coding a single pair would limit exploration. Dropdown menus for both X and Y axes let the user pivot freely — from environmental metrics to labor metrics to social attention — turning one chart into dozens of possible views.

Color Encoding

Color is mapped to a categorical field (Brand by default, switchable to Country or Sentiment Label). The Tableau-10 palette was chosen for perceptual distinctness. Color answers the “which group?” question at a glance and clusters in the scatterplot become immediately identifiable.

Side Panel and Summary Statistics

The right-hand summary panel shows aggregate statistics (mean carbon, water, wage, sustainability, landfill, ethical rating) for whatever subset is currently brushed, or the entire filtered dataset when no brush is active. This dual-level detail (overview + selection) follows Shneiderman’s “overview first, zoom and filter, details on demand” mantra.

Filters

Top-bar filters for Brand, Country, and Year range let users narrow the dataset before brushing. This prevents information overload when the user cares about a single brand or time period.

Findings

Finding 1: Production Volume and Carbon Emissions

Setting X = Production (tonnes/mo) and Y = Carbon Emissions (tCO₂e) reveals a roughly positive relationship, but with substantial scatter — some brands achieve high output with comparatively lower emissions, while others show disproportionately high emissions at moderate production levels.

Finding 2: Sustainability vs. Environmental Cost

Plotting Sustainability Score against Env Cost Index shows an expected negative trend: higher sustainability scores correlate with lower environmental cost indices. However, the relationship is noisy, suggesting that sustainability scores alone do not fully capture real environmental impact.

Finding 3: Worker Wages Across Countries

Using the Country color encoding with Avg Worker Wage on the Y-axis highlights geographic pay disparities: records from certain countries cluster at markedly lower wage levels. Brushing these clusters confirms that low-wage records also tend to have higher working hours per week.

Deployment

This project is hosted on GitHub Pages. To deploy your own copy:

  1. Create a public GitHub repository.
  2. Push all project files (including the data/ folder with the CSV).
  3. In the repository Settings → Pages, set the source to the main branch root.
  4. Your site will be live at https://<username>.github.io/<repo-name>/.
